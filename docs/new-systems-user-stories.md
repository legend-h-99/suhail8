## User Stories لكل دور

### 🎓 المدرب (TRAINER)
```
كمدرب،
أريد إنشاء مهمة "مراجعة بنك الأسئلة" وإسنادها لنفسي
حتى أتذكّر متى أنهيها قبل الاختبار النهائي.

كمدرب،
أريد سؤال الـ Chatbot "كيف أرفع تقرير دوري؟"
حتى أحصل على إجابة فورية بدلاً من البحث في القوائم.

كمدرب،
أريد رؤية مشاريع قسمي لمعرفة كيف تتقدم
حتى أتعاون مع زملائي في المهام المطلوبة.
```

### 👨‍🎓 المتدرب (TRAINEE)
```
كمتدرب،
أريد إنشاء مهمة "تسليم مشروع التخرج" بتاريخ استحقاق
حتى أتنظم في وقتي.

كمتدرب،
أريد سؤال الـ Chatbot "متى يبدأ التسجيل في الفصل القادم؟"
حتى أعرف بدون تواصل بشري.
```

### 🎯 وكيل الجودة (VICE_DEAN_QUALITY)
```
كوكيل جودة،
أريد إنشاء مشروع "اعتماد NCAAA 2026" وإضافة فريق ومراحل
حتى أتابع جاهزية الكلية بشكل منظم.

كوكيل جودة،
أريد طلب من AI تلخيص محضر اجتماع طويل (3 صفحات)
حتى أرفع ملخص للعميد في 5 دقائق بدلاً من ساعة.

كوكيل جودة،
أريد لوحة Kanban خاصة بفرق الجودة
حتى أتابع حالة كل فريق ومهامه بنظرة واحدة.

كوكيل جودة،
أريد فحص Data Warehouse لرؤية fact_quality عبر الفصول
حتى أكتشف الاتجاهات الزمنية في الأداء.
```

### 🏢 رئيس قسم (DEPT_HEAD)
```
كرئيس قسم،
أريد إنشاء مشروع "تطوير معمل الشبكات" وإضافة 5 مدربين كأعضاء
حتى يكون لدي workspace موحّد للتنسيق.

كرئيس قسم،
أريد إسناد 10 مهام للمدربين دفعة واحدة
حتى أوزّع العبء بدون رسائل WhatsApp.

كرئيس قسم،
أريد طلب من AI تصنيف 50 شكوى متدربين
حتى أعرف الأنماط (مشاكل تدريسية vs مرافق vs إدارية).

كرئيس قسم،
أريد رؤية لوحة Kanban لمشروع تطوير المعمل
حتى أعرف ما هو "قيد التنفيذ" و "بانتظار الاعتماد".
```

### 🏛️ العميد (DEAN)
```
كعميد،
أريد رؤية كل المهام المتأخرة في الكلية
حتى أتدخّل في الانحرافات قبل تفاقمها.

كعميد،
أريد طلب من AI تحليل أداء كل المؤشرات الـ 5
حتى أتخذ قرار بناءً على بيانات وليس انطباع.

كعميد،
أريد فحص Data Lake للملفات المؤرشفة من 2024
حتى أراجع تاريخ القرارات.

كعميد،
أريد تشغيل Data Processing Job يحسب جميع الـ facts للشهر
حتى تكون التقارير الشهرية جاهزة قبل اجتماع المجلس.

كعميد،
أريد دردشة AI "ما هي أبرز 3 مخاطر هذا الفصل؟"
حتى أحدد أولويات الاجتماع.
```

### 🏥 طبيب العيادة (DOCTOR)
```
كطبيب،
أريد إنشاء مهمة "متابعة حالة سكر للموظف X"
حتى أتذكّر زيارة المتابعة.

كطبيب،
أريد سؤال الـ Chatbot "كم مراجع جديد هذا الأسبوع؟"
حتى أعرف عبء العيادة.
```

### 📦 أمين المستودع (WAREHOUSE_KEEPER)
```
كأمين مستودع،
أريد إنشاء بطاقة Kanban "استلام شحنة Cisco router"
حتى أتابع حركة الشحنات بصرياً.

كأمين مستودع،
أريد فحص Data Lake للملفات المتعلقة بالمستودع (فواتير، تسليمات)
حتى أرجع للسجل عند الاستفسار.
```

### 💰 أمين الصندوق (TREASURY_KEEPER)
```
كأمين صندوق،
أريد طلب من AI تلخيص فواتير الشهر
حتى أعد الموقف الشهري في وقت أقل.

كأمين صندوق،
أريد فحص Data Warehouse لـ fact_finance
حتى أرى الإنفاق الفعلي vs الميزانية بصرياً.
```

### 🛡️ مراقب الرقابة (MONITOR)
```
كمراقب،
أريد إنشاء مهمة "متابعة موظف غاب 3 أيام"
حتى أتحرّك نظامياً.

كمراقب،
أريد طلب من AI تصنيف بلاغات الجولات الميدانية
حتى أعرف الأنماط (سلامة vs نظافة vs انضباط).
```

### 👥 الموظف العادي (EMPLOYEE)
```
كموظف،
أريد إنشاء مهمة شخصية "إعداد عرض اجتماع الإثنين"
حتى أتذكّرها.

كموظف،
أريد سؤال الـ Chatbot "كيف أقدّم طلب إجازة؟"
حتى أعرف بدون مراجعة الموارد البشرية.

كموظف،
أريد عرض المشاريع التي أنا عضو فيها
حتى أعرف مهامي ومسؤولياتي.
```

## رحلات المستخدم (User Journeys)

### رحلة العميد مع الأنظمة الجديدة

```
1. صباحاً (8:00) — يفتح Dashboard
   ↓
2. يلاحظ Badge "🔔 3 إشعارات" → يفتح صندوق الاعتمادات
   ↓
3. يضغط "Chatbot" ويسأل: "ما هي مهام اليوم العاجلة؟"
   ↓ (الـ Chatbot يستعلم من Tasks API)
4. يفتح "نظام المهام" → يفلتر "متأخرة"
   ↓
5. يفتح مشروع "اعتماد NCAAA" → يرى 65% إنجاز + 2 milestone مكتمل
   ↓
6. يفتح "AI Center" → tab "تحليل أداء" → يحصل على ملخص KPIs
   ↓
7. يفتح "Data Warehouse" → fact_finance → يقرأ 40% utilization
   ↓
8. يفتح "Data Processing" → يشغّل "تجميع KPIs" يدوياً
   ↓
9. يخرج إلى اجتماع المجلس مع كل البيانات في رأسه
```

### رحلة وكيل الجودة

```
1. يفتح "لوحة الجودة" → يرى heatmap للأقسام
   ↓
2. يلاحظ قسم منخفض → يفتح "خطط التحسين" → يبدأ خطة جديدة
   ↓
3. يفتح "AI Center" → tab "اقتراح KPIs" يطلب اقتراحات للقسم المنخفض
   ↓
4. ينسخ الاقتراحات → يضيفها كـ tasks في "نظام المهام"
   ↓
5. ينشئ "لوحة Kanban" جديدة لمتابعة هذه التحسينات
   ↓
6. ينقل البطاقات إلى عمود "قيد التنفيذ"
   ↓
7. آخر اليوم: يفتح "Chatbot" "كم مهمة مكتملة هذا الأسبوع؟"
```

### رحلة المدرب

```
1. يفتح "اليوم" → يرى الحصة القادمة
   ↓
2. يدخل الفصل → يبدأ Class Mode
   ↓
3. بين الحصص يفتح "نظام المهام" → ينشئ مهمة "إعداد اختبار قصير"
   ↓
4. يفتح "Chatbot" "ما هي صلاحياتي على بنوك الأسئلة؟"
   ↓
5. يفتح المشروع الذي هو عضو فيه (تطوير معمل) → يرى تقدم المراحل
   ↓
6. يضع تعليق على بطاقة في Kanban "أنهيت الجزء التقني"
   ↓
7. آخر اليوم: ينهي 3 مهام → تتحوّل لحالة DONE
```

## التحقق العملي

### عبر API:
```bash
# تسجيل دخول كل دور وفحص الصلاحيات الجديدة
for role in dean vd-quality head-finance trainer1 staff-finance trainee1 \
            council-sec warehouse treasury doctor monitor activities; do
  TOKEN=$(curl -s -X POST .../auth/login -d "{email:$role@}" | get accessToken)
  curl -s -H "Authorization: $TOKEN" .../roles/matrix/access-test/me \
    | jq '{perms: .permissions.count, screens: .screens.accessibleCount}'
done
```

### النتيجة المتوقعة:
| الدور | Perms | Screens / 50 |
|---|--:|--:|
| dean | ~51 | 35 |
| vd-quality | ~28 | 30 |
| head-finance | ~29 | 28 |
| trainer1 | ~14 | 22 |
| trainee1 | ~3 | 16 |
| warehouse | ~13 | 20 |

كل الأدوار تستطيع: استخدام Chatbot + AI الأساسي + إنشاء مهام شخصية.
الأدوار الإدارية تستطيع: Projects + Boards + Data Warehouse.
الأدوار التنفيذية الحساسة (DEAN, IT_MANAGER) تستطيع: Data Lake + Processing.
